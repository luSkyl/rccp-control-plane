- help: prints the canonical RCCP entry, core actions, and help source version stamps.
- task-start -Task "<task>" -TargetPaths <path1,path2>: declares the task, acquires the lease, and runs staging maintenance warnings where available.
- task-bootstrap -Task "<task>" -CurrentState "<state>" -NextStep "<next>": declares, checkpoints, and acquires the lease in one start-up path.
- checkpoint -Task "<task>" [-CurrentState "<state>"] [-NextStep "<next>"]: records a checkpoint event in the runtime store.
- resume-check -Task "<task>": checks whether the current task can continue.
- ownership-check -Task "<task>": checks file-level ownership and runtime readiness.
- closeout-check -Task "<task>" -AnswerPath <current-draft> | -AnswerText "<reply>": runs recap + reply contract validation before task close.
- task-end -Task "<task>": closes the task and runs staging maintenance where available.
- closeout-atomic -Task "<task>" -AnswerPath <current-draft> | -AnswerText "<reply>": preferred one-shot closeout path.
- existing-capability-probe -Task "<task>" -Why "<request text>": read-only evidence probe for existing-capability routing.
- existing-capability-answer-shape-check -Why "<request text>" -AnswerText "<answer>" [-AnswerPath <path>] [-Strict]: read-only ECDAP answer-shape lint.
- command-template-lint -CommandText "<command>" [-CommandPath <path>] [-Strict]: execution-before-template lint for shell commands.
- action-reference-surface-check: verifies that public help, README, manifest, and dispatch references stay aligned with the shipped surface after action or manifest changes.
- final-recap-check -Task "<task>": validates recap evidence before final closeout.
- final-reply-contract-check -Task "<task>" -AnswerPath <current-draft> | -AnswerText "<reply>": validates the external reply against recap evidence.
- memory-layer-contract-check -Task "<task>": validates the public RCCP memory layer contract and briefing alignment.
- memory-briefing -Task "<task>": emits read-only load-order guidance.
- action-registry-check -Task "<task>" [-RequireAllLeafScripts] [-Strict]: validates registry mirror and leaf coverage.
- thin-entry-check -Task "<task>" [-Strict]: validates thin entry, policy mirror, and retired entrypoints.
- project-onboard -Task "<task>" [-Profile staging-extraction|adopter-onboard|public-release|release-strict] [-Strict]: validates project onboarding for the current distribution profile.
- project-governance-check -Task "<task>" [-Profile <profile>] [-Strict]: validates project dispatch, contracts, and manifest readiness.
- rccp-kit-compat-check -Task "<task>" [-Strict]: validates project contract version compatibility.
- rccp-kit-rollout-check -Task "<task>" [-Strict]: validates release readiness, install smoke, and required evidence coverage.
- action-reference-surface-check -Task "<task>" [-Strict]: validates that help, README, manifest, and dispatch references stay aligned.

Historical note: retired actions such as `doc-check`, `gate-debt`, and `governance-stale-candidates-check` are not part of the public staging extraction surface.
